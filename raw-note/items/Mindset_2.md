### 第一章 - 無瑕的程式碼

- 概略的描述 clean code 是什麼
- 我們為什麼需要 clean code
- 不好的程式會形成阻礙, 壞的程式會導致嚴重的後果
- 讓開發速度加快的**唯一**方式是 clean code
- **更容易閱讀才會更容易撰寫**
- 童子君規則: 持續地保持 clean code, 每次 commit 都更乾淨一點點
- clean code 是
  - 十分注重細節
  - 優雅的
  - 專注表達單一意圖
  - 簡單明瞭
  - 包含測試的, unit test, acceptance test
  - 被細心照料的
  - 沒有重複的程式碼
  - 提早建立簡單抽象概念
  - 執行結果如同想像的
